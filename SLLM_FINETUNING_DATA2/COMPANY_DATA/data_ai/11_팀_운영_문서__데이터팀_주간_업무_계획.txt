## 2. 팀원별 주요 작업
### 김철수 (데이터 품질 관리)
- 데이터 오류 검토 및 수정
  - 의존성: 데이터 수집팀의 피드백 필요
  - 리스크: 데이터 수집 지연 시 작업 지연 가능성

- 품질 개선 방안 제안서 작성
  - 의존성: 데이터 분석팀의 협조 필요
  - 리스크: 제안서 승인 지연 시 적용 시점 늦어질 수 있음

### 이영희 (AI 모델 개발)
- 모델 성능 평가 및 개선
  - 의존성: 최신 데이터셋 확보 필요
  - 리스크: 데이터셋 품질 저하 시 성능 저하 우려

- 코드 리뷰 및 테스트
  - 의존성: 팀원들과의 협업 필요
  - 리스크: 코드 충돌 발생 가능성

### 박민수 (데이터 엔지니어링)
- 신규 파이프라인 설계 및 구현
  - 의존성: 기존 시스템과의 호환성 검토 필요
  - 리스크: 기술적 문제 발생 시 일정 지연 가능성

- 문서화 작업
  - 의존성: 관련 팀의 피드백 필요
  - 리스크: 문서화 지연 시 지식 공유 부족

## 4. 협업/의존성
- 데이터 수집팀: 데이터 품질 개선을 위한 피드백 제공
- 데이터 분석팀: 품질 개선 방안 제안서 검토 및 피드백
- IT팀: 신규 파이프라인 구축 시 기술 지원

## 5. 리스크/차질 대비

- 모델 성능 저하 시 추가 데이터 수집 및 재학습 계획 수립
- 파이프라인 구축 지연 시 우선순위 조정 및 리소스 재배치
