## 1. 프로젝트 개요 및 배경
본 프로젝트는 특정 비즈니스 문제를 해결하기 위해 개발된 머신러닝 모델의 성능을 평가하기 위한 보고서입니다. 데이터/AI팀은 모델의 정확성과 신뢰성을 확보하기 위해 다양한 성능 지표를 활용하여 평가를 진행하였습니다. 이 보고서는 모델의 성능을 검증하고 향후 개선 방향을 제시하는 것을 목표로 합니다.

## 2. 모델/알고리즘 범위
- **모델 종류**: Random Forest, XGBoost
- **목표**: 이진 분류 (예: 긍정/부정)
- **데이터 출처**: 내부 데이터베이스 및 공개 데이터셋
- **주요 변수**: 특성 변수 및 레이블

## 3. 실험/테스트 절차
### 데이터셋
- **훈련 데이터**: 70%
- **검증 데이터**: 15%
- **테스트 데이터**: 15%

### 환경
- **프로그래밍 언어**: Python
- **라이브러리**: scikit-learn, XGBoost
- **하드웨어**: GPU/CPU 혼합 환경

### 하이퍼파라미터
- **Random Forest**: n_estimators=100, max_depth=10
- **XGBoost**: learning_rate=0.1, n_estimators=100, max_depth=6

### 체크리스트
- [ ] 데이터 전처리 완료
- [ ] 모델 학습 및 검증 완료
- [ ] 하이퍼파라미터 튜닝 완료

## 4. 성능 지표 및 기준
- **정확도 (Accuracy)**: 전체 예측 중 맞춘 비율
- **F1 점수**: 정밀도와 재현율의 조화 평균
- **ROC-AUC**: 수신자 조작 특성 곡선 아래 면적
- **혼동 행렬**: TP, TN, FP, FN 수치 확인

### 기준
- Accuracy ≥ 0.85
- F1 score ≥ 0.80
- ROC-AUC ≥ 0.85

### 체크리스트
- [ ] 성능 지표 계산 완료
- [ ] 기준 만족 여부 확인

- **검증 포인트**:

  - [ ] 통계적 유의성 검증 (p-value 확인)

## 6. 리스크/한계 및 개선 제안
### 리스크
- **데이터 편향**: 특정 클래스의 데이터가 불균형할 경우 성능 저하 가능
- **과적합**: 모델이 훈련 데이터에 과도하게 적합할 위험

### 한계
- **모델 해석성 부족**: 복잡한 모델일수록 해석이 어려움
