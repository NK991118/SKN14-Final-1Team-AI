# 민감 데이터 접근 키·암호화 키 관리 문서

## 1. 개요 및 배경
민감 데이터 접근 키와 암호화 키는 기업의 정보 보안 핵심 요소입니다. 이 문서는 CodeNova의 민감 데이터 보호 및 키 관리 전략을 정립하고, 잠재적 위협에 대한 대응 방안을 제시합니다. 

## 2. 위협 시나리오 및 영향
- **무단 접근**: 내부자 또는 외부 공격자가 키를 획득하여 민감 데이터에 접근할 수 있음.
- **키 유출**: 키가 유출될 경우, 데이터의 기밀성과 무결성이 위협받을 수 있음.
- **키 관리 실패**: 키의 생성, 저장, 폐기 과정에서의 오류로 인해 보안 취약점 발생.

## 3. 대응 전략 (단계별 실행)
### 3.1 키 생성 및 저장
- 안전한 알고리즘 사용: AES, RSA 등 검증된 암호화 알고리즘 사용.
- 키 생성 후 즉시 안전한 저장소(예: HSM, KMS)에 저장.

### 3.2 접근 제어
- 최소 권한 원칙 적용: 필요한 인원에게만 접근 권한 부여.
- 접근 로그 기록 및 주기적 검토.

### 3.3 키 회전 및 폐기
- 정기적인 키 회전 정책 수립: 최소 6개월마다 키 변경.
- 사용하지 않는 키는 즉시 안전하게 폐기.

### 3.4 교육 및 인식 제고
- 직원 대상 보안 교육 실시: 민감 데이터와 키 관리의 중요성 인식.

## 4. 검증/점검 체크리스트
- [ ] 키 생성 및 저장 방식 검토.
- [ ] 접근 권한 관리 현황 점검.
- [ ] 키 회전 및 폐기 기록 검토.
- [ ] 보안 교육 이수 현황 확인.

## 5. 리스크 한계 및 보완책
- **리스크 한계**: 키 유출 및 무단 접근의 가능성.
- **보완책**:
  - 다중 인증(MFA) 도입: 키 접근 시 추가 인증 절차 요구.
  - 정기적인 보안 감사 실시: 외부 전문가에 의한 보안 점검.

## 6. CTO 보고·승인 포인트
