# 데이터베이스 스키마 & 변경 이력 문서
분류: backend | 회사: 코드노바 | 버전: v1.0 | 작성일: 2025-08-29

## 1. 데이터베이스 스키마 개요

코드노바의 데이터베이스는 주요 서비스의 기능을 지원하기 위해 설계되었습니다. 데이터베이스는 다음과 같은 주요 엔티티로 구성됩니다:

- **사용자(User)**: 사용자 정보를 저장합니다.
- **글쓰기(Article)**: 생성된 글의 정보를 저장합니다.
- **이미지(Image)**: 업로드된 이미지 정보를 저장합니다.
- **요약(Summary)**: 생성된 요약 정보를 저장합니다.
- **챗봇 대화(Chat)**: 챗봇과의 대화 기록을 저장합니다.
- **광고(Ads)**: 대화형 광고 정보를 저장합니다.

각 엔티티는 서로 관계를 가지며, 데이터의 무결성을 유지하기 위해 외래 키 제약 조건을 설정합니다.

## 2. 데이터베이스 스키마 상세

### 2.1 사용자(User)

- **user_id**: 사용자 고유 ID
- **username**: 사용자 이름
- **email**: 이메일 주소
- **password**: 비밀번호 (해시값 저장)
- **created_at**: 가입 일자
- **updated_at**: 정보 수정 일자

### 2.2 글쓰기(Article)

- **article_id**: 글 고유 ID
- **user_id**: 작성자 ID (사용자 테이블과 연결)
- **title**: 글 제목
- **content**: 글 내용
- **created_at**: 작성 일자
- **updated_at**: 수정 일자

### 2.3 이미지(Image)

- **image_id**: 이미지 고유 ID
- **user_id**: 업로더 ID (사용자 테이블과 연결)
- **url**: 이미지 URL
- **created_at**: 업로드 일자

### 2.4 요약(Summary)

- **summary_id**: 요약 고유 ID
- **article_id**: 관련 글 ID (글쓰기 테이블과 연결)
- **summary_text**: 요약 내용
- **created_at**: 생성 일자

### 2.5 챗봇 대화(Chat)

- **chat_id**: 대화 고유 ID
- **user_id**: 사용자 ID (사용자 테이블과 연결)
- **message**: 대화 내용
- **created_at**: 대화 일자

### 2.6 광고(Ads)

- **ad_id**: 광고 고유 ID
- **user_id**: 광고주 ID (사용자 테이블과 연결)
- **ad_content**: 광고 내용
- **created_at**: 생성 일자

## 3. 변경 이력

변경 이력은 데이터베이스 스키마의 변경 사항을 기록하여 추적할 수 있도록 합니다. 변경 사항이 있을 경우 아래와 같은 형식으로 기록합니다.

### 3.1 변경 이력 기록 양식

- **변경 날짜**: YYYY-MM-DD
- **변경 사항**: 구체적인 변경 내용
- **작성자**: 변경을 수행한 개발자 이름

### 3.2 예시

- **변경 날짜**: 2025-08-01
- **변경 사항**: 사용자 테이블에 'last_login' 필드 추가
- **작성자**: 홍길동

## 4. 검증 포인트

데이터베이스 스키마 변경 후 다음 사항을 검증합니다:

- 모든 테이블과 필드가 올바르게 생성되었는지 확인
- 외래 키 제약 조건이 제대로 설정되었는지 확인
- 기존 데이터와의 호환성 검증
- 쿼리 성능 테스트 수행

## 5. 결론

본 문서는 코드노바의 데이터베이스 스키마 및 변경 이력에 대한 기본 정보를 제공합니다. 향후 변경 사항이 있을 경우, 반드시 변경 이력을 기록하고 검증 포인트를 확인하여 데이터의 무결성을 유지해야 합니다.
